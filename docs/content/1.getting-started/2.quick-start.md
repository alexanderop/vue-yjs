# Quick Start

vue-yjs uses Vue's provide/inject pattern to share a `Y.Doc` across your component tree. Here's how to build a simple collaborative counter.

::steps

### Provide a Y.Doc

In your root or layout component, call `useProvideYDoc` to create and share a document:

```vue
<!-- App.vue -->
<script setup lang="ts">
import { useProvideYDoc } from 'vue-yjs'

// Creates a new Y.Doc and provides it to all descendants
// Automatically destroyed when the component unmounts
const doc = useProvideYDoc()
</script>

<template>
  <Counter />
</template>
```

### Use shared types in child components

Any component in the tree can inject the doc and bind to shared types:

```vue
<!-- Counter.vue -->
<script setup lang="ts">
import { useYMap } from 'vue-yjs'

// Injects the Y.Doc automatically, binds to the "counter" map
const { data, set } = useYMap<{ count: number }>('counter', {
  defaults: { count: 0 }
})
</script>

<template>
  <button @click="set('count', data.count + 1)">
    Count: {{ data.count }}
  </button>
</template>
```

That's it! The `data` ref updates reactively whenever the underlying Yjs map changes -- whether from this component, another component, or even a remote client.

### Add networking (optional)

To sync across clients, add a WebSocket provider:

```vue
<!-- App.vue -->
<script setup lang="ts">
import { useYRoom } from 'vue-yjs'

const { doc, status, synced } = useYRoom('my-room', {
  serverUrl: 'wss://demos.yjs.dev/ws',
})
</script>

<template>
  <p>Status: {{ status }} | Synced: {{ synced }}</p>
  <Counter />
</template>
```

`useYRoom` combines `useProvideYDoc`, `useWebSocketProvider`, and optionally `useIndexedDB` into a single composable.

::

## How it works

1. `useProvideYDoc()` creates a `Y.Doc` and provides it via Vue's dependency injection (using `provideLocal`)
2. Composables like `useYMap` call `useYDoc()` internally to inject the doc
3. Reactive state is backed by `shallowRef` -- updates only trigger when Yjs fires `observeDeep`
4. Cleanup is automatic via `onScopeDispose` -- works in components, `effectScope`, and `watchEffect`
