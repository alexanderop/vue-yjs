# useUndoManager

Reactive wrapper around `Y.UndoManager`. Creates an undo manager for the given scope and exposes reactive `canUndo` / `canRedo` computed refs that update whenever the stacks change. The undo manager is destroyed when the enclosing effect scope is disposed.

::tip
Adjust `captureTimeout` to control undo granularity. A lower value (e.g. `200`) creates finer-grained undo steps; a higher value (e.g. `1000`) groups more edits together.
::

## Usage

```vue
<script setup lang="ts">
import { useYText, useUndoManager, useProvideYDoc } from 'vue-yjs'

useProvideYDoc()
const { text, replace, yText } = useYText('editor')
const { undo, redo, canUndo, canRedo } = useUndoManager(yText)
</script>

<template>
  <textarea :value="text" @input="replace(($event.target as HTMLTextAreaElement).value)" />
  <div>
    <button :disabled="!canUndo" @click="undo">Undo</button>
    <button :disabled="!canRedo" @click="redo">Redo</button>
  </div>
</template>
```

Tracking multiple types:

```ts
const yText = doc.getText('editor')
const yMeta = doc.getMap('metadata')

// Track changes to both types in a single undo manager
const { undo, redo } = useUndoManager([yText, yMeta])
```

## Parameters

::field-group
  ::field{name="scope"}
  `Y.AbstractType<any> | Y.AbstractType<any>[]` — One or more Yjs shared types to track.
  ::

  ::field{name="options" type="UseUndoManagerOptions"}
  Optional configuration (see Options below).
  ::
::

## Options

::field-group
  ::field{name="captureTimeout" type="number"}
  Time in milliseconds during which edits are merged into a single undo step. Default: `500`.
  ::

  ::field{name="trackedOrigins"}
  `Set<unknown>` — Origins whose changes the undo manager should track. By default, all local changes are tracked.
  ::
::

## Return Value

::field-group
  ::field{name="undo"}
  `() => void` — Undo the last captured change.
  ::

  ::field{name="redo"}
  `() => void` — Redo the last undone change.
  ::

  ::field{name="canUndo"}
  `ComputedRef<boolean>` — Whether there are changes to undo.
  ::

  ::field{name="canRedo"}
  `ComputedRef<boolean>` — Whether there are changes to redo.
  ::

  ::field{name="stopCapturing"}
  `() => void` — Force a new capture group for the next edit.
  ::

  ::field{name="clear"}
  `() => void` — Clear both the undo and redo stacks.
  ::

  ::field{name="undoManager" type="Y.UndoManager"}
  The underlying `Y.UndoManager` instance.
  ::
::

## Capture Timeout

The `captureTimeout` option controls how edits are grouped. Within the timeout window, consecutive edits are merged into a single undo step. Call `stopCapturing()` to force the next edit into a new group regardless of timing.
