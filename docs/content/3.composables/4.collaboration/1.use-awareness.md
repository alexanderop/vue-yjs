# useAwareness

Reactive binding for the Yjs awareness protocol. Tracks remote and local awareness states in a shallow ref that updates on every `"change"` event. The listener is removed automatically when the enclosing effect scope is disposed.

::note
Awareness state is ephemeral — it is automatically removed when a client disconnects. Use shared types (`Y.Map`, `Y.Array`) for persistent collaborative data.
::

## Usage

```vue
<script setup lang="ts">
import { useAwareness, useWebSocketProvider, useProvideYDoc } from 'vue-yjs'

interface Cursor {
  x: number
  y: number
  name: string
  color: string
}

const doc = useProvideYDoc()
const { awareness: awarenessInstance } = useWebSocketProvider(
  'wss://demos.yjs.dev/ws', 'my-room', doc
)

const { states, localClientId, setLocalState, setLocalStateField } =
  useAwareness<Cursor>(awarenessInstance)

// Set your presence
setLocalStateField('name', 'Alice')
setLocalStateField('color', '#42b883')
</script>

<template>
  <div
    v-for="[clientId, cursor] of states"
    :key="clientId"
    class="cursor"
    :style="{ left: cursor.x + 'px', top: cursor.y + 'px', color: cursor.color }"
  >
    {{ cursor.name }}
  </div>
  <p>Connected clients: {{ states.size }}</p>
  <p>My client ID: {{ localClientId }}</p>
</template>
```

::field-group

## Type Parameter

::field-group
  ::field{name="T"}

  ::
::

## Return Value

::field-group
  ::field{name="states"}
  `Readonly<ShallowRef<Map<number, T>>>` — Reactive map of all awareness states keyed by client ID.
  ::

  ::field{name="localClientId" type="number"}
  The local client's ID in the awareness protocol.
  ::

  ::field{name="setLocalState"}
  `(state: Partial<T> | null) => void` — Replace the entire local awareness state, or pass `null` to clear it.
  ::

  ::field{name="setLocalStateField"}
  `<K extends keyof T>(field: K, value: T[K]) => void` — Update a single field on the local awareness state.
  ::
::

## Awareness vs Shared Types

Use awareness for ephemeral state that:
- Doesn't need conflict resolution (latest-write-wins)
- Should disappear when a client disconnects
- Updates frequently (cursors, selections)

Use shared types (`Y.Map`, etc.) for persistent state that:
- Needs CRDT conflict resolution
- Should survive disconnections
- Is part of the document
