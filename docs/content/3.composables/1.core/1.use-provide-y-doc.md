# useProvideYDoc

Provides a `Y.Doc` instance to the current and all descendant components via Vue's dependency injection. If no document is passed, a new one is created and automatically destroyed when the effect scope is disposed.

::tip
When no `doc` is passed, the created document is automatically destroyed on scope dispose — no manual cleanup needed.
::

## Usage

```vue
<script setup lang="ts">
import { useProvideYDoc } from 'vue-yjs'

// Create and provide a new doc (auto-destroyed on unmount)
const doc = useProvideYDoc()
</script>
```

Providing an existing doc (caller manages lifecycle):

```ts
import * as Y from 'yjs'
import { useProvideYDoc } from 'vue-yjs'

const externalDoc = new Y.Doc()
useProvideYDoc(externalDoc)
// externalDoc is NOT destroyed on scope dispose
```

## Parameters

::field-group
  ::field{name="doc" type="Y.Doc"}
  An existing `Y.Doc` to provide. When omitted, a new document is created and owned by this scope.
  ::
::

## Return Value

::field-group
  ::field{name="Returns" type="Y.Doc"}
  The provided (or newly created) `Y.Doc` instance.
  ::
::

## Lifecycle

- **No argument**: Creates a new `Y.Doc` and destroys it on `onScopeDispose`
- **With argument**: Provides the given doc without managing its lifecycle — the caller is responsible for destroying it

## Type Declarations

```ts
import type { Y } from 'yjs'

declare function useProvideYDoc(doc?: Y.Doc): Y.Doc
```
