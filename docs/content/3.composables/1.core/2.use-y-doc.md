# useYDoc

Returns a `Y.Doc` — either one passed directly or one injected from the current or an ancestor component. Works transparently in the **same** component as `useProvideYDoc` (useful in Nuxt composable patterns).

::warning
Throws an error at runtime if no `doc` argument is provided and no ancestor (or same-component) `useProvideYDoc` call exists.
::

## Usage

```vue
<script setup lang="ts">
import { useYDoc } from 'vue-yjs'

// Inject the doc provided by useProvideYDoc
const doc = useYDoc()
const yMap = doc.getMap('settings')
</script>
```

Same-component pattern (common in Nuxt composables):

```ts
// composables/useCollaboration.ts
export function useCollaboration() {
  const doc = useProvideYDoc()
  // ...
}

// composables/useTodoList.ts
export function useTodoList() {
  const doc = useYDoc() // works in the same component!
  // ...
}
```

## Parameters

::field-group
  ::field{name="doc" type="Y.Doc"}
  Optional `Y.Doc` to use directly. If provided, skips injection.
  ::
::

## Return Value

::field-group
  ::field{name="Returns" type="Y.Doc"}
  The resolved `Y.Doc` instance.
  ::
::

## Errors

Throws an `Error` if no `doc` was passed and no ancestor or same-component call to `useProvideYDoc` exists.

## Exports

vue-yjs also exports the injection key:

```ts
import { YDocKey } from 'vue-yjs'
// YDocKey: InjectionKey<Y.Doc>
```

## Type Declarations

```ts
import type { InjectionKey } from 'vue'
import type { Y } from 'yjs'

declare const YDocKey: InjectionKey<Y.Doc>

declare function useYDoc(doc?: Y.Doc): Y.Doc
```
