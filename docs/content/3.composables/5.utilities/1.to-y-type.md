# toYType

Recursively converts a plain JavaScript value to the corresponding Yjs shared type. Used internally by `useYMap`, `useYArray`, and other composables to handle automatic conversion when setting values.

::note
You rarely need to call `toYType` directly — composables like `useYMap` and `useYArray` handle conversion automatically.
::

## Usage

```ts
import { toYType } from 'vue-yjs'
import * as Y from 'yjs'

// Objects → Y.Map
const yMap = toYType({ name: 'Alice', age: 30 })
// yMap is a Y.Map with keys 'name' and 'age'

// Arrays → Y.Array
const yArray = toYType([1, 2, 3])
// yArray is a Y.Array with items 1, 2, 3

// Nested structures
const yNested = toYType({
  users: [
    { name: 'Alice', tags: ['admin'] }
  ]
})
// Y.Map → { users: Y.Array → [ Y.Map → { name: 'Alice', tags: Y.Array → ['admin'] } ] }

// Primitives pass through
toYType('hello')     // 'hello'
toYType(42)          // 42
toYType(true)        // true
toYType(null)        // null

// Already a Yjs type → pass through
const existing = new Y.Map()
toYType(existing)    // same instance, unchanged
```

## Conversion Rules

| Input | Output |
|-------|--------|
| Plain object (`{}`) | `Y.Map` with recursively converted values |
| Array (`[]`) | `Y.Array` with recursively converted items |
| `Y.AbstractType` (any Yjs type) | Pass through unchanged |
| `Uint8Array` | Pass through unchanged |
| `string`, `number`, `boolean`, `null` | Pass through unchanged |

## When to Use

You typically don't need to call `toYType` directly — the typed composables (`useYMap`, `useYArray`) handle conversion automatically when you call their `set`, `push`, or `insert` methods.

Use `toYType` directly when you need to:
- Work with the raw `Y.Map` or `Y.Array` API
- Convert complex nested structures before inserting into a Yjs type

```ts
const yArray = doc.getArray('items')

// Manual conversion needed when using raw Yjs API
yArray.push([toYType({ name: 'New Item', tags: ['important'] })])
```
