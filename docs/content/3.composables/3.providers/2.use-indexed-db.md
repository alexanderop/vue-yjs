# useIndexedDB

Reactive wrapper around `y-indexeddb`'s `IndexeddbPersistence`. Persists the given `Y.Doc` to the browser's IndexedDB so that document state survives page refreshes. Exposes a reactive `synced` ref that becomes `true` once the stored content has been loaded.

::warning
IndexedDB may be unavailable in private/incognito browsing mode. Always provide an `onError` callback to handle failures gracefully.
::

## Usage

```vue
<script setup lang="ts">
import { useProvideYDoc, useIndexedDB } from 'vue-yjs'

const doc = useProvideYDoc()
const { synced } = useIndexedDB('my-document', doc)
</script>

<template>
  <span v-if="!synced">Loading from cache...</span>
</template>
```

With error handling and sync callback:

```ts
const { whenSynced, clearData } = useIndexedDB('my-document', doc, {
  onError: (err) => console.warn('[IndexedDB]', err),
  onSynced: (persistence) => console.log('Loaded from cache!'),
})

// Wait for cached data to load
await whenSynced
```

## Parameters

::field-group
  ::field{name="docName" type="string" required}
  A unique name used as the IndexedDB database name.
  ::

  ::field{name="doc" type="Y.Doc" required}
  The `Y.Doc` to persist.
  ::

  ::field{name="options" type="UseIndexedDBOptions"}
  Optional configuration (see Options below).
  ::
::

## Options

::field-group
  ::field{name="onError"}
  `(error: unknown) => void` — Error callback for IndexedDB failures (private browsing, quota exceeded, DB corruption). Default: `console.error`.
  ::

  ::field{name="onSynced"}
  `(persistence: IndexeddbPersistence) => void` — Callback fired once when persisted content is loaded into the doc.
  ::
::

## Return Value

::field-group
  ::field{name="persistence" type="IndexeddbPersistence"}
  The underlying `IndexeddbPersistence` instance.
  ::

  ::field{name="synced"}
  `Readonly<ShallowRef<boolean>>` — Whether the local IndexedDB content has been loaded into the doc.
  ::

  ::field{name="whenSynced"}
  `Promise<IndexeddbPersistence>` — Promise that resolves once persisted content is loaded.
  ::

  ::field{name="clearData"}
  `() => Promise<void>` — Clear all persisted data from IndexedDB for this document.
  ::

  ::field{name="destroy"}
  `() => Promise<void>` — Destroy the persistence instance. Safe to call multiple times.
  ::
::

## Lifecycle

The persistence is created immediately and destroyed on `onScopeDispose`. The `synced` event listener is also cleaned up automatically.

## Peer Dependencies

This composable requires `y-indexeddb`:

```bash
pnpm add y-indexeddb
```
